## โปรแกรม Go แรกของคุณ

ใน TypeScript/Node.js คุณสร้างไฟล์, export หรือเขียนโค้ดไว้ที่ top-level แล้ว `node` ก็รันให้ ส่วน Go กำหนดให้ทุกโปรแกรมที่รันได้ต้องมีโครงสร้างเฉพาะ: ประกาศ `package main` และมี entry point `func main()` มาเทียบกัน

export const tsHelloCode = `// hello.ts
console.log("Hello, World!");

// รันด้วย: npx ts-node hello.ts
// หรือคอมไพล์: tsc hello.ts && node hello.js`;

export const goHelloCode = `// hello.go
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}

// รันด้วย: go run hello.go`;

<TsGo client:visible ts={tsHelloCode} go={goHelloCode} />

## อธิบายทีละบรรทัด

**`package main`**

ทุกไฟล์ Go เริ่มด้วยการประกาศ `package` และ package ชื่อ `main` นั้นพิเศษ — มันบอกว่านี่คือโปรแกรมที่รันได้ (ไม่ใช่ library) คิดเสียว่าเทียบเท่ากับไฟล์ที่ field `"main"` ใน `package.json` ชี้ไปหา

**`import "fmt"`**

การ import ของ Go เป็นแบบ explicit และทำทีละไฟล์ `fmt` คือ standard library package สำหรับงาน formatted I/O — เทียบได้กับ `console` ในโลก Go คุณ import ด้วย full package path แบบ string (ต้องใส่ quote) และ import ที่ไม่ได้ใช้จะกลายเป็น compile error

**`func main()`**

Function `main` คือ entry point ทุกโปรแกรม Go ที่รันได้ต้องมี `func main()` เพียงตัวเดียวใน package `main` และมันไม่รับ argument — ถ้าต้องการ command-line argument ให้เข้าถึงผ่าน `os.Args`

export const tsEntryCode = `// TypeScript: หลาย entry style
// 1. Top-level code (scripts)
console.log("รันทันที");

// 2. "main" ใน package.json ชี้มาที่นี่
// 3. Exported function ที่ framework เรียก`;

export const goEntryCode = `// Go: entry point เดียวเสมอ
package main

import "fmt"

func main() {
    // โปรแกรมเริ่มที่นี่เสมอ
    fmt.Println("รันก่อน")
}`;

<TsGo client:visible ts={tsEntryCode} go={goEntryCode} />

## fmt.Println vs console.log

`fmt.Println` คือสิ่งที่ใกล้เคียง `console.log` ที่สุด นอกจากนี้ package `fmt` ยังมี `fmt.Printf` สำหรับ format string (เหมือน `printf` ใน C หรือ template literal ใน JS) และ `fmt.Sprintf` สำหรับสร้าง string โดยไม่ print ออกมา

export const tsLogCode = `// TypeScript logging
console.log("Hello");                    // ง่ายๆ
console.log("Name:", name);             // หลาย arg
console.log(\`Score: \${score}\`);         // template literal
const msg = \`Hello, \${name}!\`;          // สร้าง string`;

export const goFmtCode = `// Go fmt package
fmt.Println("Hello")                    // ง่ายๆ
fmt.Println("Name:", name)              // หลาย arg
fmt.Printf("Score: %d\\n", score)        // format verb
msg := fmt.Sprintf("Hello, %s!", name)  // สร้าง string`;

<TsGo client:visible ts={tsLogCode} go={goFmtCode} />

## Format verb

`fmt.Printf` ใช้ format verb แทน template literal ตัวที่ใช้บ่อยมีดังนี้:

| Verb | ความหมาย | เทียบเท่า TypeScript |
|------|---------|---------------------|
| `%s` | string | `${str}` |
| `%d` | integer | `${num}` |
| `%f` | float | `${num.toFixed(6)}` |
| `%.2f` | float, ทศนิยม 2 ตำแหน่ง | `${num.toFixed(2)}` |
| `%v` | ค่าใดๆ (รูปแบบ default) | `${JSON.stringify(val)}` |
| `%T` | ชื่อ type | `typeof val` |
| `%+v` | struct พร้อมชื่อ field | `JSON.stringify(val, null, 2)` |

<GoOnly title="Import ที่ไม่ได้ใช้คือ compile error">

ใน TypeScript import ที่ไม่ได้ใช้จะเป็นแค่ warning (หรือไม่เป็นอะไรเลย ขึ้นกับ tsconfig) แต่ใน Go import ที่ไม่ได้ใช้คือ **compile error** ข้อบังคับนี้ทำให้ไฟล์ Go สะอาดอยู่เสมอ แต่ก็อาจสะดุดได้ตอนคุณกำลังลองแก้โค้ดเล่น

```go
import (
    "fmt"
    "os"   // ERROR: imported and not used
)

func main() {
    fmt.Println("hello")
    // os ถูก import แต่ไม่ได้ใช้ — จะไม่คอมไพล์
}
```

วิธีแก้: ลบ import นั้นออก หรือใช้ blank identifier `_` เพื่อทำเป็น side-effect import (เจอไม่บ่อย ใช้เฉพาะเวลาต้องการ side effect จาก `init()` เท่านั้น):

```go
import _ "github.com/lib/pq" // side-effect: register pq driver
```

Editor สำหรับ Go ส่วนใหญ่ (VS Code + gopls) จะลบ import ที่ไม่ได้ใช้ให้อัตโนมัติตอน save

## ลองเลย — รันโปรแกรม Go แรกของคุณ

export const playgroundHelloCode = `package main

import "fmt"

func main() {
	fmt.Println("Hello, World!")

	// fmt.Printf ใช้ format verb — เหมือน template literal
	name := "Gopher"
	age := 3
	fmt.Printf("Name: %s, Age: %d\\n", name, age)

	// fmt.Sprintf สร้าง string โดยไม่ print
	greeting := fmt.Sprintf("Welcome, %s!", name)
	fmt.Println(greeting)
}`;
