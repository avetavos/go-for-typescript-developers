## ตั้งค่า environment Go ของคุณ

การตั้งค่า TypeScript ต้องใช้ Node.js, npm (หรือ package manager อื่น) และ `tsc` ส่วน Go มี toolchain เดียวแบบรวมศูนย์ — ติดตั้ง Go ครั้งเดียวได้ครบทุกอย่าง: compiler, formatter, test runner และ module manager

## ติดตั้ง Go

ดาวน์โหลด official installer จาก [go.dev/dl](https://go.dev/dl/) หลังติดตั้ง ตรวจสอบด้วย:

```bash
go version
# go version go1.22.0 darwin/arm64
```

โดยค่าเริ่มต้น Go จะติดตั้งที่ `/usr/local/go` ส่วน `GOPATH` (workspace directory, ค่าเริ่มต้นคือ `~/go`) คือที่เก็บ module ที่ดาวน์โหลดมาและ binary ที่ติดตั้งไว้ คุณแทบไม่ต้องสนใจมันเลย — `go mod` จัดการให้หมด

## Go CLI vs Node/npm CLI

นักพัฒนา Node.js ทุกคนรู้จัก `node`, `npm` และ `tsc` ส่วน Go รวมทุกอย่างนี้ไว้ใน `go` command เดียว

export const tsCliCode = `# TypeScript / Node.js toolchain
node index.js            # รัน file
npx ts-node src/index.ts # รัน TypeScript โดยตรง
tsc                      # คอมไพล์ TypeScript
npm install              # ติดตั้ง dependency
npm run build            # รัน build script
npx <tool>              # รัน package binary`;

export const goCliCode = `# Go toolchain (ทั้งหมดใน command เดียว)
go run main.go           # คอมไพล์ + รันใน step เดียว
go run .                 # รัน package ปัจจุบัน
go build -o myapp .      # คอมไพล์เป็น binary
go mod init github.com/you/myapp  # init module ใหม่
go get github.com/gin-gonic/gin   # เพิ่ม dependency
go install tool@latest   # ติดตั้ง CLI tool แบบ global`;

<TsGo client:visible ts={tsCliCode} go={goCliCode} />

## go run vs go build

`go run` คือ development loop ของคุณ — คอมไพล์แล้วรันในขั้นตอนเดียว เหมือน `ts-node` หรือ `npx tsx` ที่ใช้ตอน develop ส่วน `go build` สร้าง binary แบบ standalone ที่เอาไป deploy ได้ Go ไม่มี "watch mode" ในตัว แต่เครื่องมืออย่าง [air](https://github.com/air-verse/air) ช่วยทำ live reload ได้

export const tsRunCode = `# TypeScript development loop
npx ts-node src/index.ts    # คอมไพล์ + รัน (dev)
tsc && node dist/index.js   # คอมไพล์แล้วรัน (CI/prod)
npx tsx watch src/index.ts  # watch mode`;

export const goRunCode = `# Go development loop
go run .                    # คอมไพล์ + รัน (dev)
go build -o ./bin/app . && ./bin/app  # คอมไพล์แล้วรัน
# air                       # watch mode (third-party)`;

<TsGo client:visible ts={tsRunCode} go={goRunCode} />

## go mod init — เทียบเท่ากับ npm init

เมื่อเริ่มโปรเจกต์ Go ใหม่ ให้ initialize module ด้วย `go mod init` คำสั่งนี้จะสร้างไฟล์ `go.mod` — เทียบเท่ากับ `package.json` ของ Go

export const tsNpmInitCode = `# TypeScript
npm init -y
# สร้าง package.json พร้อม name, version, deps

# แล้วเพิ่ม dependency:
npm install express
# อัพเดท package.json + สร้าง package-lock.json`;

export const goModInitCode = `# Go
go mod init github.com/yourname/myapp
# สร้าง go.mod

# แล้วเพิ่ม dependency:
go get github.com/gin-gonic/gin
# อัพเดท go.mod + สร้าง go.sum (เหมือน package-lock.json)`;

<TsGo client:visible ts={tsNpmInitCode} go={goModInitCode} />

## go install — CLI tool แบบ global

`npm install -g` ใช้ติดตั้ง CLI tool แบบ global ส่วน `go install` ก็ทำแบบเดียวกัน — คอมไพล์ module แล้ววาง binary ไว้ใน `$GOPATH/bin` (อย่าลืมเพิ่ม path นี้ใน `PATH` ของคุณ)

export const tsInstallCode = `# TypeScript: CLI tools แบบ global
npm install -g typescript
npm install -g prettier
npx create-react-app myapp`;

export const goInstallCode = `# Go: CLI tools แบบ global
go install golang.org/x/tools/gopls@latest      # LSP server
go install github.com/air-verse/air@latest       # live reload
go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest`;

<TsGo client:visible ts={tsInstallCode} go={goInstallCode} />

<GoOnly title="GOPATH, GOROOT, และ module cache">

คุณจะเห็นการอ้างถึง environment variable เหล่านี้อยู่เป็นระยะ:

| Variable | ค่าเริ่มต้น | คืออะไร |
|----------|------------|---------|
| `GOROOT` | `/usr/local/go` | ตำแหน่งที่ติดตั้ง Go toolchain |
| `GOPATH` | `~/go` | workspace ของคุณ — module cache อยู่ที่ `~/go/pkg/mod/` |
| `GOBIN` | `$GOPATH/bin` | ตำแหน่งที่ `go install` วาง binary |

รัน `go env` เพื่อดู environment variable ของ Go ทั้งหมด หรือ `go env GOPATH` เพื่อดูทีละค่า คุณแทบไม่ต้องแก้ค่าพวกนี้เลย — `go mod` ทำให้ module cache โปร่งใสอยู่แล้ว

```bash
# diagnostics ที่มีประโยชน์
go env          # แสดง Go environment variable ทั้งหมด
go env GOPATH   # แสดงแค่ GOPATH
go version      # แสดง Go version ที่ติดตั้ง
go doc fmt      # อ่าน docs ของ fmt package ในเทอร์มินัล
```
